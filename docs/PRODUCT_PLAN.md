# TheVIA - 제품 기획서

## 1. 프로젝트 비전 및 목표

### 비전
"VIA JSON을 한 번만 올리면, 모두가 쓴다."

VIA(usevia.app)는 QMK 기계식 키보드의 키 리매핑 표준 도구이지만, 공식 저장소에 미등록된 3모드(유선/BT/2.4G) 키보드 사용자들은 JSON 정의 파일을 직접 구해서 수동 로드해야 한다. TheVIA는 커뮤니티 기반 JSON 자동매칭 시스템을 통해 이 진입장벽을 제거한다.

### 핵심 목표
1. **진입장벽 제거**: 키보드를 연결하면 커뮤니티 DB에서 JSON이 자동 매칭되어, 초보자도 바로 키 설정 가능
2. **커뮤니티 신뢰 시스템**: 사용 이력과 투표 기반으로 JSON 품질을 자동 검증
3. **VIA 완전 호환**: 기존 VIA의 모든 기능(키맵, 매크로, 라이팅, 인코더)을 100% 유지
4. **오픈소스 생태계 기여**: GPL-3.0 라이선스 유지, 커뮤니티 중심 운영

### 성공 지표
| 지표 | 목표 (런칭 6개월) |
|------|-------------------|
| 커뮤니티 등록 JSON 수 | 500+ 키보드 모델 |
| 월간 활성 사용자(MAU) | 5,000+ |
| JSON 자동매칭 성공률 | 연결 시도의 70%+ |
| 평균 신뢰도 점수 | 등록 JSON의 80%가 신뢰도 "높음" |

## 2. 타겟 사용자

### 주요 타겟 (Primary)
| 페르소나 | 특성 | 핵심 니즈 |
|----------|------|-----------|
| **VIA 초보 사용자** | 키보드는 샀는데 VIA JSON 구하는 법을 모름 | 연결하면 바로 동작 |
| **3모드 키보드 사용자** | 유무선 겸용 키보드 (Nut65, RK, Epomaker 등) | 공식 미등록 JSON 자동 적용 |
| **중국산 기계식 키보드 사용자** | 타오바오/알리 구매, VIA 지원 표기되어 있지만 공식 미등록 | JSON 파일 검색 없이 바로 사용 |

### 보조 타겟 (Secondary)
| 페르소나 | 특성 | 핵심 니즈 |
|----------|------|-----------|
| **키보드 커뮤니티 기여자** | JSON을 직접 만들거나 수정할 수 있는 중급+ 사용자 | 기여 채널, 기여에 대한 인정 |
| **키보드 제조사/판매자** | 자사 키보드의 VIA 지원을 쉽게 제공하고 싶음 | JSON 등록을 통한 고객 편의 제공 |

### 사용 환경 제약
- **브라우저**: Chrome, Edge, Opera 등 Chromium 기반 (WebHID API 필수)
- **연결**: USB 유선 또는 2.4G 동글 (블루투스는 WebHID 미지원으로 불가)
- **OS**: Windows, macOS, Linux, ChromeOS

## 3. 핵심 기능 정의

### 3.1 기존 VIA 기능 (완전 유지)

| 기능 | 설명 |
|------|------|
| 키맵 리매핑 | 레이어별 키 배정 변경 (실시간 반영) |
| 매크로 | 키 시퀀스, 딜레이 포함 매크로 설정 |
| 라이팅 | RGB 매트릭스, 백라이트 설정 |
| 인코더 | 로터리 인코더 CW/CCW/Press 설정 |
| 레이아웃 옵션 | 스플릿 스페이스바 등 물리적 레이아웃 선택 |
| JSON 수동 로드 | Design 탭에서 사용자가 직접 JSON 파일 업로드 |

### 3.2 커뮤니티 JSON 자동매칭 시스템 (신규 핵심)

#### 3.2.1 매칭 로직
```
키보드 연결
  --> WebHID로 VID + PID + productName 읽기
  --> vendorProductId = VID * 65536 + PID 계산
  --> 1차: VIA 공식 정의 검색 (기존 로직)
  --> 2차: 커뮤니티 DB 검색 (VID + PID + productName 조합)
  --> 매칭 결과에 따른 UX 분기
```

#### 3.2.2 매칭 결과별 UX

| 상황 | 동작 |
|------|------|
| 공식 VIA 등록 키보드 | 기존 VIA와 동일하게 자동 연결 (Official 배지) |
| 커뮤니티 JSON 1개 (신뢰도 높음) | 자동 연결, 알림 없음 (Community 배지) |
| 커뮤니티 JSON 1개 (신뢰도 낮음) | 자동 연결 + 상단 알림바 표시 (확인/투표 UI) |
| 커뮤니티 JSON 2개+ | 신뢰도 최고 자동 매칭 + 드롭다운으로 대체 JSON 전환 |
| 매칭 없음 | JSON 업로드 안내 UI + 커뮤니티 기여 유도 |

#### 3.2.3 VID/PID/productName 매칭 규칙
- **매칭 키**: `VID + PID + productName` 조합
- **유선 vs 동글**: 동일 키보드라도 연결 방식에 따라 VID/PID가 다르므로 별도 매칭 레코드
- **VID/PID 충돌**: 다른 키보드지만 동일 VID/PID인 경우 productName으로 분리, 추가 구분 불가 시 드롭다운 선택

### 3.3 신뢰도(Trust) 시스템

#### 신뢰도 점수 구성

| 신호 | 유형 | 점수 영향 | 설명 |
|------|------|-----------|------|
| 정상 세션 완료 | 암묵적(자동) | +1 | JSON 로드 -> 설정 작업 -> JSON 교체 없이 종료 |
| 추천(Upvote) | 명시적(수동) | +3 | 사용자가 직접 추천 버튼 클릭 |
| 비추천(Downvote) | 명시적(수동) | -5 | 사용자가 비추천 버튼 클릭 |
| JSON 교체 | 암묵적(자동) | -2 | 자동매칭 JSON을 다른 JSON으로 교체 |
| 최초 업로드 | - | 초기값 0 | 업로드 즉시 매칭 가능, 단 낮은 신뢰도 표시 |

#### 신뢰도 등급

| 등급 | 조건 | UX 표시 |
|------|------|---------|
| Verified | 점수 20+ | 녹색 배지, 알림 없음 |
| Trusted | 점수 10~19 | 녹색 배지, 알림 없음 |
| Unverified | 점수 1~9 | 노란색 배지, 상단 알림바 |
| New | 점수 0 | 회색 배지, 상단 알림바 (확인 요청) |
| Flagged | 점수 음수 | 빨간색 배지, 자동매칭 제외 (수동 선택만 가능) |

### 3.4 JSON 업로드 및 기여 시스템

| 기능 | 설명 |
|------|------|
| 익명 업로드 | 계정 없이 JSON 업로드 가능 (기여 진입장벽 최소화) |
| 자동 검증 | 업로드 시 JSON 스키마 유효성 + VIA 정의 포맷 검증 |
| 중복 감지 | 동일 VID/PID/name에 동일 JSON 이미 존재 시 알림 |
| 업로드자 식별 | 선택적 닉네임 또는 브라우저 fingerprint 기반 식별 |

### 3.5 추가 제안 기능

#### Phase 1 포함
| 기능 | 설명 |
|------|------|
| 연결 방식 표시 | USB(유선)/동글 구분 표시 (사용자 혼란 방지) |
| JSON 미리보기 | 업로드 전 키보드 레이아웃 렌더링 미리보기 |

#### Phase 2 포함
| 기능 | 설명 |
|------|------|
| JSON 버전 관리 | 같은 키보드의 JSON 업데이트 시 버전 이력 유지 |
| 키보드 검색/브라우징 | 커뮤니티 등록 키보드 목록 검색 (연결 없이) |
| 제조사 인증 배지 | 제조사가 직접 등록한 JSON에 Official Vendor 배지 |

#### Phase 3 포함
| 기능 | 설명 |
|------|------|
| 키맵 프리셋 공유 | 인기 키맵 설정을 커뮤니티에서 공유/다운로드 |
| 다국어 지원 | VIA 기본 i18n + 한국어/중국어 우선 |
| 통계 대시보드 | 인기 키보드, 활성 기여자, 매칭 성공률 등 |

### 6.1 경쟁 환경 분석

| 서비스 | 특징 | 한계 |
|--------|------|------|
| **VIA (usevia.app)** | 공식 표준, 안정적 | 공식 등록 키보드만 자동 지원. 미등록 키보드는 JSON 수동 로드 필요 |
| **Vial (get.vial.today)** | VIA 포크, 자체 펌웨어 | 별도 펌웨어(Vial) 필요. VIA 펌웨어와 비호환 |
| **QMK Configurator** | 웹 기반 키맵 설정 | 펌웨어 컴파일 필요. 실시간 변경 불가 |
| **제조사 자체 SW** | 제조사별 전용 앱 | 해당 제조사 키보드만 지원. 설치 필요 |

### 6.2 TheVIA 차별화

| 차별화 포인트 | 설명 |
|---------------|------|
| **Zero-config 경험** | 연결 즉시 커뮤니티 JSON 자동매칭 -- 초보자도 JSON이 뭔지 몰라도 사용 가능 |
| **커뮤니티 기반 신뢰 시스템** | 중앙 관리 없이 사용자 행동 데이터로 JSON 품질 자동 검증 |
| **VIA 100% 호환** | 기존 VIA 펌웨어 그대로 사용, 추가 펌웨어 설치 불필요 |
| **3모드 키보드 특화** | 유선/동글 VID/PID 분리 관리로 3모드 키보드 완벽 대응 |
| **기여 진입장벽 제로** | 계정 없이 JSON 업로드 가능, 사용하는 것만으로도 신뢰도 기여 |
| **네트워크 효과** | 사용자가 늘수록 커버되는 키보드가 늘고, 키보드 커버리지가 늘수록 사용자 유입 증가 |

### 6.3 해자(Moat)

- **데이터 네트워크 효과**: 커뮤니티 JSON DB는 사용자가 쌓을수록 가치가 높아지며, 이를 복제하기 어려움
- **VIA 생태계 호환**: VIA 공식이 변경되어도 포크 기반이므로 추적 업데이트 가능
- **오픈소스 신뢰**: GPL-3.0으로 투명한 운영, 커뮤니티 기여 유도
